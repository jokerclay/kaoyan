# 进制 浮点数 字符型
## 进制变换
* x86 架构是 小端存储， 低位在前，高位在后  00 00 00 7b
* 为什么内存数据要用 十六 进制去看 ? 就是非常高效,
    * 1 位 => 1 bit 存储 0 或 1
    * 1 字节 => 1 byte = 8 bite
    * 1 Kb => 1024 byte
    * 1 Mb => 1024 Kb
    * 1 Gb => 1024 Mb

    0x 00 00 00 00  -->  0x ff ff ff ff
    2 的 32 次方 -> 1024 1024 1024 4 => 4Gb 寻址空间
